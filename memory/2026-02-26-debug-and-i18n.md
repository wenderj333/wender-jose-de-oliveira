## Critical Bug Fixes (Mural Upload & Display)

### 1. Upload Error: "FetchEvent.respondWith returned an error: Returned response is null"
- **Initial Diagnosis:** Suspected `frontend/vercel.json` rewrite rule blocking `/api` calls.
- **First Fix Attempt (Commit `b61e420`):** Modified `frontend/vercel.json` to allow `/api` calls by adding a specific rewrite for `/api/(.*)`. Deployed.
- **Result:** Error persisted (user reported it again).
- **Second Fix Attempt (Commit `64b8d7e`):** Created a Vercel serverless proxy function at `frontend/api/[...slug].js` to handle all `/api` requests and proxy them to the backend, hoping to bypass `FetchEvent` issues. Deployed.
- **Result:** Upload status pending user re-test.

### 2. Visual Issues on Mural Page
- **User Report:**
  - Wrong background (patterned green instead of dark mode).
  - Images not loading (broken camera icons).
- **Diagnosis:**
  - **Background:** `frontend/src/styles/main.css` had `.mural-grid-container { background: #fafbfc; }` overriding inline dark mode style in `MuralGrid.jsx`.
  - **Images:** `getAvatarUrl` and `getMediaUrl` in `MuralGrid.jsx` had a faulty fallback (using non-existent `/default-avatar.jpg` and potentially incorrect `API_BASE` resolution).
- **Fix Attempt (Commit `19bf003`):**
  - Changed `main.css` to `background: #0f0f1a;` for `.mural-grid-container`.
  - Updated `getAvatarUrl` to use `https://via.placeholder.com/44?text=User` as a robust fallback.
  - Updated `getMediaUrl` with clearer logic for `API_BASE` and direct URLs.
- **Result (Commit `e3c31f7`):** ✅ **Deployed successfully.** Build passed, commit and push complete. Awaiting user re-test to confirm visual fixes.
