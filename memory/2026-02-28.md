# 2026-02-28 - OpenClaw Cron Bug Fix

## Issue
Cron job to send welcome messages to new Sigo com Fé users was failing with 500 error.
- Found 17 new users ✅
- Failed to send welcome messages ❌
- API endpoint returning full user objects instead of IDs (misleading error message)

## Root Cause
Mismatch between endpoint response format and script expectations:
- **Endpoint** (`backend/src/openclaw.js`): Returned array directly
  ```json
  [{ "id": 1, "email": "...", "created_at": "..." }]
  ```
- **Script** (`send_welcome_messages.py`): Expected wrapped object
  ```python
  new_users_data = api_response_data.get('users', [])  # ← Always empty!
  ```

## Fix Applied
Modified `/api/openclaw/users/new` endpoint to wrap response:
```javascript
// Before:
res.json(r.rows);

// After:
res.json({ users: r.rows });
```

**Commit**: `4f785b9` — "Fix: Wrap users/new response in object with 'users' key for correct API parsing"
**Pushed**: ✅ to master
**Status**: Render will redeploy in 2-3 minutes

## Next Run
Cron will retry on next scheduled execution. The 17 users are still in the system — they'll receive welcome messages now that the endpoint returns the correct format.
